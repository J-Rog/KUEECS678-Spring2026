Installing QEMU

For today's lab, you will be using the QEMU VM which you can install on your local machine. It can either be Linux, Windows or MacOS. It is also available on the lab machines.

  2. Then, go to the Downloads page and follow the instructions for your specific Operating System. Download QEMU.

Adding a System Call to the xv6 Kernel
We will be modifying the sysproc.c, defs.h, syscall.h, syscall.c, user.h, and usys.S files To add a system call to the kernel source directory carry out the following steps:
  1. We first begin by modifying the sysproc.c file. Open it using 
    vim sysproc.c +
    and add the following bit of code to the end of the file 
                   int
                sys_hello(void)
                {
                  cprintf("hello_world\n");
                  return 0;
                }
    (Screenshot)

  2. Next exit sysproc.c using :wq and enter defs.h using 
    vim defs.h
    locate the comment that says proc.c and at the bottom of the table add 
    int         sys_hello(void);
    (Screenshot)
  3. Exit defs.h :wq and navigate to syscall.h 
    vim syscall.h
    At the bottom of the file add the line 
    #define SYS_hello   22
    (Screenshot)
  4. Next we register the new syscall in the syscall dispatcher. Open syscall.c 
    vim syscall.c
    Find where the other system calls are, they are labeled with the keyword extern and add our own system call 
    extern int sys_hello(void);
    (Screenshot)
    We also need to add our entry so navigate down to the sys_call table under the extern declarations and add our system call in the table using 
    [SYS_hello]   sys_hello,
    (Screenshot)
  5. Exit syscall.c using :wq and navigate to user.h 
    vim user.h
    In there we need to add our system call to a table that users in user space have access to 
    int hello(void);
    (Screenshot)
  6. Finally exit user.h and enter usys.S 
    vim usys.S +
    At the bottom add the following command for the assembly stub 
    SYSCALL(hello)
    (Screenshot)

    7. With this last modification you've successfully completed all the steps necessary to change the kernel of an operating system. Next lets look at how to modify the kernel compilation so your changes show up in the xv6.
